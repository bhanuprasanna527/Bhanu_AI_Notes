Types of Supervised Learning

Supervised learning can be further divided into two types of problems:

  1. Regression
  2. Classification

Regression

Regression algorithms are used if there is a relationship between the input variable and the output variable. It is used for the prediction of continuous variables, such as Weather forecasting, Market Trends, etc. Below are some popular Regression algorithms which come under supervised learning:

  1. Linear Regression
  2. Regression Trees
  3. Non-Linear Regression
  4. Bayesian Linear Regression
  5. Polynomial Regression

Classification

Classification algorithms are used when the output variable is categorical, which means there are two classes such as Yes-No, Male-Female, True-false, etc.

  1. Random Forest
  2. Decision Trees
  3. Logistic Regression
  4. Support Vector Machines

To summarize Regression: Regression is used to predict a number infinitely many possible outputs.

To summarize Classification: Classification predicts categories of small number of possible outputs.

Unsupervised Learning

Unsupervised learning is when there is no labelling of data to get the output.

Unsupervised learning is used to get the patterns and structures from the given output.

Unsupervised learning cannot apply a regression or classification problem because unlike supervised learning, we have the input data but no corresponding output data. The goal of unsupervised learning is to find the underlying structure of dataset, group that data according to similarities, and represent that dataset in a compressed format.

The unsupervised learning algorithm can be further categorized into two types of problems:

Unsupervised algorithm has to find structure in the data

Some of the types of Unsupervised Learning:

  1. Clustering
  2. Association
  3. Dimensionality Reduction
  4. Auto Encoders (Deep learning)

Clustering algorithm is an unsupervised algorithm. Clustering is a process of seperating different patterns that can be grouped into a specific set of points which can be visualized as a cluster.

    Learning Rate

  • Learning rate is a hyperparameter that is it is intialised by the developer. Learning rate plays a important role in Model training. So, Initializing the learning rate must be done carefully.
  • In order for gradient descent to work, we must set a good learning rate.
  • This parameter determines how fast or slow we move towards optimal weights.
  • If the Learning rate is high or very high we might skip the optimal solution.
  • If the Learning rate is small or too small we migth reach the optimal solution slowly which reduces the efficiency of the algorithm or solution.
  • Reaching a good Learning rate takes time and Trail and error method to find a optimal learning rate for further models or algorithms.
  • In simple language, we can define learning rate as how quickly our network abandons the concepts it has learned up until now for new ones.
