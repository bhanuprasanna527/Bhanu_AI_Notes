Machine Learning Diagnostic

Diagnostic: A A test that you run to gain insight into what is/isn’t working with a learning algorithm, to gain guidance into improving its performance.

Diagnostics can take time to implement but doing so can be a very good use of your time.

This week we will learn more about the methods to evaluate and conduct diagnosis on machine learning models.

Evaluating a Model

Evaluating a Model is required so as to test the models validity with other data.

There can be a model that can be fitted with training data that when new data comes in the model the loss increases. This happens when the data is overfitted that it increases the overall loss and increases the cost of the model.

So, one way is to split the data into Train/Test split.

  • The splitting of dataset can be best for Evaluating a Model. This is a practice that all ML developers use.

    • Model Selection

    Model Selection is way of selecting a model which is most appropriate for the data that is collected.

    Model Selection is selecting the best model out of the created models. We have to create various versions of model using various parameter tuning.

  • An example,

    • - Let us take polynomial regression, where we will develop 10 models with varying level of polynomial like 1st degree, 2nd degree, ....., 10th degree polynomial models.

    - We have to split the data into 2 types: Train set and Test set. Suppose, after seeing the cost J(w,b) we can see that 5th order polynomial is performing better it doesn't fully mean it can satisfy other data when given. It means that the model does not generalize well.

    - So, we have to introduce another set of data making the split of data into 3 parts: Train set, Cross Validation set (sometimes referred to as Devlopment set or Dev set or Validation Set), and Test set. We first Train the models using Train set, and use the Dev set for validating the model that is it enables us to select the model before fully testing it. It enables us to select the model appropriate for use can be selected after viewing at Train loss and Validation loss. So, it will make us be optimistic about Model Testing with Test set.

    Bias and Variance

    In this section we will discuss about Bias and Variance.

    Bias vs Variance

    Bias is found when the model is Underfit, Variance is found when the model is Overfit.



  • As we can see in the above image, we can see that Bias is present when the model underfit that is it does not understand the data well. Variance is present when the model overfit that is it understands the data more perfectly than needed.
  • Train error and Validation error is high in High Bias state. Train error is low and Validation error is high in High Variance state. In Just Right case, the train error and validation error is less.
  • There can also be case where the model has both Bias and Variance. This happens when the model predicts the data perfectly for certain region (High Variance) and it predicts the data wrongly (High Bias). Normally, this case appears very rarely.

    • Regularization and Bias/Variance

    Regularization injected cost function for Linear Regression in Course 1:

    $$J(\mathbf{w},b) = \frac{1}{2m} \sum\limits_{i = 0}^{m-1} (f_{\mathbf{w},b}(\mathbf{x}^{(i)}) - y^{(i)})^2 + \frac{\lambda}{2m} \sum_{j=0}^{n-1} w_j^2$$

    Here the λ term can be varyed. If the λ is very large it reduces the parameters weight in the equation where the model is Underfitted. If λ is small it makes the model to be susceptible to Overfitting which we don't want. The below diagram illustrates this point.

    Bias/Variance and Neural Network

    In Neural Network Bias and Variance play an important role in deciding the appropriate model.

    Bias-Variance Trade-off is term most commonly used by Machine Learning Practitioners. It means that the model selected is an appropriate model that has been obtained where the model selected is "Just Right".



    Neural Network and Regularization

    Regularization plays an important role in Neural Network. In order to explain let us take an example:




  • Here we can see in the image that there is a small network and a large network. When small network applies Regularization then it comes in par with a large network which signifies the importance of Regularization in neural networks.

    • Implementation of Regularization in Tensorflow

    Let us look at the implementation of MNIST model with and without Regularization.

    Unregularized MNIST model

    
layer_1 = Dense(units=25, activation=”relu")
layer_2 = Dense(units=15, activation=”relu")
layer_3 = Dense(units=1, activation="sigmoid")
model = Sequential([layer_1, layer_2, layer_3])

    Regularized MNIST model

    
layer_1 = Dense(units=25, activation=”relu”, kernel_regularizer=L2(0.01))
layer_2 = Dense(units=15, activation=”relu”, kernel_regularizer=L2(0.01))
layer_3 = Dense(units=1, activation="sigmoid”, kernel_regularizer=L2(0.01))
model = Sequential([layer_1, layer_2, layer_3])

    Spam Classification Example

    Here we will discuss about the spam classification example. Spam is an email that is often for spoofing and phishing scams.

    Often spam email contain spelling mistakes that can be a way to recognize Spam mail.


    Building a Spam Classifier


    Supervised learning: x = features of email
               y = spam (1) or not spam (0)

    Features: list the top 10,000 words to compute 𝑥1, 𝑥2, ⋯ , 𝑥10,000

    How to try to reduce your spam classifier’s error?
  • Collect more data. E.g., “Honeypot” project.
  • Develop sophisticated features based on email routing (from email header).
  • Define sophisticated features from email body. E.g., should “discounting” and ”discount” be treated as the same word.
  • Design algorithms to detect misspellings. E.g., w4tches, med1cine, m0rtgage.

    • Error Analysis

    Error Analysis is a phase of the Iterative Loop. Here, we can first train the model and find the total error arised by different examples by the model.

  • Let us take a example, Spam Classification, suppose there are 500 examples as Cross Validation set provided to the model for Cross Validation error.
  • Then we suppose get 100 examples as error or was not able to predict correctly. We can go to each individual example and categorize them based on common traits.
  • This step cannot be possible if there are more examples like 10000 and the error examples might be about 1000 which you cannot parse through each one of them and categorize them.
  • So, one of the best method is to categorize a part of the error examples in our case it is 1000 out of which we categorize about 100 samples.

    • Adding Data

    Adding more data is a step mostly practiced by ML Practitioners in order to improve the overall model performance and try to reduce Variance in the model. As we have already discussed that Variance can be reduced by adding more data.

    There are two ways of creating Data:

  • Data Augmentation
  • Data Synthesis

    • Data Augmentation

    Augmentation: modifying an existing training example to create a new training example.

    Data Augmentation is done most commonly for image and speech or sound data.

    Data Augmentation involves the process of distorting the data. Here is picture of cute cat to demonstrate Data Augmentation:



  • Here we have a same cat picture with different angles and rotations.
  • These are only some methods of Data Augmentation, there are many methods like changing contrast, brightness, cropping, zoom in, zoom out, and many others.

    • Data Synthesis

    Suppose you are working in a new model for developing a OCR (Optical Character Recognition) where a model will identify all the text present in the image.




    The above image has a lot of text present. We have train a model that can understand all types of fonts, rotation, and angles of text to identify all the text present in the image.

    This is where the Data Synthesis comes in where we create the data or engineer the data from scratch using a word editor and putting different types of fonts and rotating to create a dataset for training of the model.


    Transfer Learning

    Transfer learning is a machine learning method where a model developed for a task is reused as the starting point for a model on a second task.

    In Transfer learning we take the parameters of already trained models and try to change accordingly to our like for our prediction.

    There can be 2 options for Transfer learning:

  • Only train output layers parameters.
  • Train all parameters.

    • Transfer Learning Summary:

  • Download neural network parameters pretrained on a large dataset with same input type (e.g., images, audio, text) as your application (or train your own).
  • Further train (fine tune) the network on your own data.

    Fairness, Bias and Ethics

    Let us discuss about Bias and Ethics individually:

    Bias

    Bias is when the model created shows some irregularities associated:

  • Hiring tool that discriminates against women.
  • Facial recognition system matching dark skinned individuals to criminal mugshots.
  • Biased bank loan approvals.
  • Toxic effect of reinforcing negative stereotypes.

    • Adverse use cases

    Some of the most recent Adverse Use cases:

  • Deepfake technology: This model created a video of President Obama face being mimicked and the lip movements of a movie dialogue. This type of model is harm to the society and it can affect various factors like economics, politics, etc..
  • Spreading toxic/incendiary speech through optimizing for engagement.
  • Generating fake content for commercial or political purposes.
  • Using ML to build harmful products, commit fraud etc.

    Guidelines

    Let us discuss some guidelines:

  • Get a diverse team to brainstorm things that might go wrong, with emphasis on possible harm to vulnerable groups.
  • Carry out literature search on standards/guidelines for your industry.
  • Audit systems against possible harm prior to deployment (After the 3rdstep i.e Training the model in the above diagram).
  • Develop mitigation plan (if applicable), and after deployment, monitor for possible harm.

    • Skewed Datasets

    Skewed dataset is the scenario where the number of observations belonging to one class are significantly lower than those belonging to other class.

    On skewed datasets (e.g., when there are more positive examples than negative examples, accuracy is not a good measure of performance and you should instead use F score, which is based on precision and recall.

    Error Metrics

    In order to understand Error Metrics, we have to first understand Confusion Matrix which is base for all error metrics.

    Let us see the Confusion Matrix Diagram to understand better.




    Precision

    Precision tells us about, what fraction is actually true?


    Recall

    Recall tells us about, what fraction did we correctly detect which are true?




    F1 - Score

    By calculating the harmonic mean of a classifier's precision and recall, the F1-score combines both into a single metric. It mainly serves to compare the effectiveness of two classifiers. Assume classifiers A and B have higher recall and precision, respectively. The F1 scores for both classifiers in this situation can be used to determine which one yields superior results.
